# BeaconLink Code of Conduct

## Our Commitment

BeaconLink is committed to providing a welcoming, respectful, and collaborative environment for everyone.

We believe that healthy technical communities are built on professionalism, mutual respect, open discussion, and constructive collaboration. Every participant deserves to be treated with dignity regardless of:

- Age
- Body size
- Disability
- Ethnicity
- Gender identity or expression
- Level of experience
- Nationality
- Personal appearance
- Race
- Religion
- Sexual orientation
- Technical background

Participation in the BeaconLink community requires adherence to this Code of Conduct.

---

# Our Standards

Examples of behavior that contribute to a positive environment include:

- Being respectful and professional
- Giving constructive feedback
- Welcoming newcomers
- Sharing knowledge openly
- Being patient with questions
- Assuming good intentions
- Focusing on technical discussions rather than personal disagreements
- Accepting feedback gracefully
- Respecting different viewpoints
- Helping improve documentation and code quality

Professional disagreement is expected and encouraged when supported by evidence and technical reasoning.

---

# Unacceptable Behavior

The following behaviors are not acceptable:

- Harassment or intimidation
- Personal attacks
- Discriminatory language or behavior
- Hate speech
- Threats of violence
- Deliberate trolling
- Repeated disruptive behavior
- Publishing another person's private information without permission
- Sexualized language or imagery unrelated to the project
- Spam or repeated off-topic promotion
- Knowingly submitting malicious code
- Deliberately disrupting project discussions

Unacceptable behavior applies across all project spaces, including:

- GitHub Issues
- Pull Requests
- Discussions
- Documentation
- Community chat platforms
- Project events

---

# Technical Discussions

BeaconLink values thoughtful technical debate.

Contributors are encouraged to:

- Explain design decisions.
- Support arguments with evidence.
- Discuss trade-offs objectively.
- Critique ideas rather than individuals.
- Remain open to alternative solutions.

Strong technical disagreement is acceptable.

Personal hostility is not.

---

# Community Responsibilities

Community members are expected to:

- Treat others respectfully.
- Keep discussions productive.
- Report inappropriate behavior.
- Respect maintainers' decisions.
- Help improve the project.

Maintainers are expected to model these behaviors consistently.

---

# Maintainer Responsibilities

Project maintainers are responsible for:

- Enforcing this Code of Conduct fairly and consistently.
- Moderating discussions.
- Addressing reported incidents promptly.
- Protecting community members from harassment.
- Explaining moderation decisions when appropriate.
- Acting in the best interests of the community.

Maintainers may remove, edit, or reject contributions that violate this Code of Conduct.

---

# Reporting Issues

If you experience or witness behavior that violates this Code of Conduct, please report it to the project maintainers.

When submitting a report, include:

- A description of the incident
- Relevant links or screenshots
- Dates and times (if applicable)
- Any additional context that may assist the investigation

Reports will be handled as confidentially as reasonably possible.

---

# Enforcement

Project maintainers may take any action they consider appropriate in response to unacceptable behavior.

Possible actions include:

1. Private warning
2. Public warning
3. Temporary suspension from community participation
4. Removal of comments or contributions
5. Permanent ban from the project

The severity of the response will depend on the nature, frequency, and impact of the behavior.

---

# Scope

This Code of Conduct applies within all official BeaconLink community spaces, including:

- GitHub repositories
- GitHub Issues
- Pull Requests
- Discussions
- Documentation
- Project websites
- Official communication channels

It also applies when an individual is representing the project in public.

---

# Conflict Resolution

Most disagreements can be resolved through respectful discussion.

Contributors are encouraged to:

- Clarify misunderstandings before escalating.
- Focus on facts rather than assumptions.
- Seek consensus where practical.
- Accept project decisions once made.

Not every technical discussion will end with unanimous agreement.

---

# Enforcement Principles

Enforcement decisions should be:

- Fair
- Consistent
- Proportionate
- Transparent where appropriate
- Focused on protecting the community

The primary goal of enforcement is maintaining a healthy and productive project environment.
