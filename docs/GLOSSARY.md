# BeaconLink Glossary

This glossary defines the terminology used throughout the BeaconLink
documentation.

Unless explicitly stated otherwise, the definitions in this document are
considered authoritative for all BeaconLink architecture, design,
implementation, and operational documentation.

---

# A

## Agent

A software component installed on a managed node that maintains a secure
outbound connection to an BeaconLink Relay.

The Agent is responsible for:

- Runtime management
- Deployment execution
- Health reporting
- Configuration reconciliation
- Inventory collection
- Local observability

---

## API Gateway

The public entry point for BeaconLink APIs.

The API Gateway authenticates requests, applies authorization policies,
routes traffic to internal services, and exposes REST and WebSocket
interfaces.

---

## Artifact

A versioned deployment package consumed by the Deployment Engine.

Examples include:

- Container images
- Virtual machine images
- WebAssembly modules
- Binary packages

---

## Audit Log

An immutable record of security-sensitive or operational events
generated by the BeaconLink platform.

Audit records support compliance, troubleshooting, and forensic
analysis.

---

# B

## Bootstrap

The initial process of registering a new Agent with the BeaconLink Control
Plane.

Bootstrap establishes identity, trust, and secure communication.

---

# C

## Configuration Drift

A condition where the observed infrastructure state differs from the
desired state declared within a Site Manifest.

BeaconLink continuously detects and reconciles drift.

---

## Control Plane

The collection of BeaconLink services responsible for management,
orchestration, policy evaluation, deployment planning, and operational
coordination.

The Control Plane does not execute workloads directly.

---

# D

## Data Plane

The distributed infrastructure responsible for executing workloads and
carrying operational traffic.

Agents, runtimes, and deployed applications belong to the Data Plane.

---

## Declarative Infrastructure

An infrastructure management approach where operators define the desired
end state rather than the sequence of actions required to achieve it.

BeaconLink continuously reconciles actual infrastructure toward this desired
state.

---

## Deployment

The process of reconciling desired application state onto managed
infrastructure.

Deployments are immutable and versioned.

---

## Desired State

The target infrastructure configuration described within a Site
Manifest.

BeaconLink continuously attempts to reconcile actual infrastructure toward
this state.

---

# E

## Event

An immutable record describing something that has already occurred
within the platform.

Examples include:

- DeploymentCompleted
- AgentConnected
- PolicyEvaluated

---

## Event Streaming Platform

The internal event backbone used by BeaconLink services for asynchronous
communication.

---

# F

## Federation

The coordination of multiple BeaconLink Relays or sites into a single logical
management environment.

---

# G

## GitOps

An operational model where Git repositories serve as the authoritative
source of declarative infrastructure configuration.

---

# H

## Health Check

A mechanism used to determine whether a managed component is operating
correctly.

---

# I

## Immutable Deployment

A deployment strategy in which new application versions replace existing
versions rather than modifying them in place.

---

# M

## Manifest

A declarative configuration document describing desired infrastructure.

Within BeaconLink, the Site Manifest is the primary manifest type.

---

## Multi-Site

A deployment topology spanning multiple independent infrastructure
locations managed by a single BeaconLink installation.

---

# P

## Plugin

An extension that adds functionality to BeaconLink through a stable extension
API without modifying the core platform.

---

## Policy

A declarative rule evaluated by the Policy Engine to determine whether a
requested operation is permitted.

---

## Policy Engine

The service responsible for evaluating governance and operational
policies throughout the BeaconLink platform.

---

# R

## Reconciliation

The continuous process of comparing observed infrastructure with desired
state and applying changes to eliminate differences.

---

## Relay

A secure communication gateway between BeaconLink Agents and the Control
Plane.

Relays terminate outbound Agent connections and forward authenticated
management traffic.

---

## Runtime

An execution environment capable of running workloads.

Examples include:

- Containers
- Virtual machines
- Kubernetes
- WebAssembly

---

## Runtime Provider

An implementation of the Runtime Abstraction Layer supporting a
particular execution platform.

---

# S

## Scheduler

The platform service responsible for executing time-based and recurring
operations.

---

## Secret Provider

A component responsible for securely retrieving secrets from BeaconLink or an
external secret management system.

---

## Site

A logical grouping of infrastructure managed as a single operational
unit.

A site may represent:

- Data center
- Cloud region
- Edge location
- Kubernetes cluster
- Office
- Manufacturing facility

---

## Site Manifest

The declarative document describing the desired state of a Site.

---

# T

## Tenant

The highest logical ownership boundary within a multi-tenant BeaconLink
deployment.

Each tenant owns its own resources, users, policies, deployments, and
configuration.

---

# W

## Workflow

A declarative sequence of coordinated operational activities executed by
the Workflow Engine.

---

## Workflow Engine

The orchestration service responsible for coordinating long-running
multi-step operations across BeaconLink services.

---

# Z

## Zero Trust

A security model in which every connection, identity, request, and
operation must be explicitly authenticated and authorized regardless of
network location.
